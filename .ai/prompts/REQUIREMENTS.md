# 요구사항 정의 문서

## 개발 원칙
1. **단순함 유지**: 불필요한 복잡성을 피하고 직관적인 기능을 제공합니다.
2. **코드 일관성**: Kotlin 컨벤션과 Material Design 가이드를 준수합니다.

## 프로젝트 개요
- **이름**: gitnote
- GitHub 저장소를 기반으로 하는 안드로이드 마크다운 메모장 앱.
- 주요 특징: 메모 CRUD, 로컬 Room DB와 마크다운 파일의 양방향 연동, 다중 GitHub 레포지토리 지원.

## 기술 요구 사항
- Android SDK 36 지원 (minSdk 24)
- Material Design 3 (다크 모드 기본 지원)
- Room Persistence Library (로컬 메타데이터 저장)
- GitHub REST API 연동 (다중 레포지토리 및 보안 인증)
- 마크다운 렌더링 (Markwon) 및 이미지 로딩 (Coil)

## 디자인 요구 사항
- Material Design 기반의 모던한 UI
- 라이트/다크 모드 완벽 지원
- 상단 툴바를 통한 상태 정보(현재 레포지토리 이름) 제공

## 기능 요구 사항
### 1. 메모 관리
- **데이터 구조**: 메모 1개는 Room DB 레코드 1개이자 마크다운 파일 1개로 관리됨.
- **파일 명명**: 랜덤 문자열로 파일명 생성.
- **메타데이터**: 생성/수정 시간을 마크다운 헤더(Front Matter)에 기록 및 자동 업데이트.
- **이미지**: 본문 내 다중 이미지 삽입 지원 (마크다운 문법 사용).

### 2. 목록 화면 (MainActivity)
- **정렬/검색**: 날짜순, 제목순 정렬 및 제목/본문 통합 검색.
- **미리보기**: 제목, 본문 3줄 미리보기, 마지막 수정 시간 표시.
- **복사 기능**: 롱클릭을 통해 메모 복사 (제목 앞에 'copy' 접두어 추가).
- **일괄 삭제**: 다중 선택 모드를 통한 메모 일괄 삭제 (삭제 전 확인 팝업 필수).

### 3. 뷰어 및 편집기
- **읽기 전용 모드**: 마크다운 프리뷰 제공, 상단바에 생성일/수정일 표시.
- **편집 모드**: 텍스트 편집 및 실시간 이미지 삽입, 상단바에 "Edit Mode" 표시.

### 4. GitHub 연동 (설정)
- **보안**: GitHub 액세스 키(PAT 등)는 소스코드나 앱 내 일반 저장소에 노출되지 않도록 보안 수준을 높여 관리.
- **다중 레포지토리**: 여러 개의 레포지토리를 등록하고 설정에서 전환 가능.
- **상태 표시**: 현재 선택된 레포지토리 이름을 목록 상단에 고정 표시.
- **동기화**: 목록 화면에서 직접 Pull/Push 버튼을 통해 동기화 수행.
- **브랜치**: `main` 또는 `master` 브랜치 단일 사용.

## 미확정 사항 (결정 시 기능 요구 사항으로 이동)
- 이미지 외부 저장소(S3, 클라우드 등) 사용 여부.
- 이미지 참조 시점의 권한 관리 상세 로직.
