# AI Prompt Guidelines (작업 지침서, 작업 지시서)

## 최우선 사항
- 모든 AI 질문, 응답, 주석 작업, 커밋 로그 등등 **전반에 걸쳐서** 되도록 **한국어** 를 사용한다
- **터미널 LLM 대답도 한국어를 사용한다**
- 단 파일명은 영어, 코드 관례 처럼 영어를 사용해야 하는 곳은 영어를 사용한다

## 주의 사항 (LLM 이 자주 실 수 하는 것)
- `.ai` 폴더 하위의 문서들을 읽은 후 문서를 의도한 대로 읽었는지 테스트 해보는 방법
  1. 한국어로 대답하는가 ?
     - 아주 간단하고, 소스코드와 무관한 작업을 하나 실행해서 테스트 진행해야 함
  2. 작업 1개를 완료하면, `커밋 + 알람 스크립트 실행` 을 매번 잘 하는가 ?
     - 더미 텍스트 파일을 미리 만들어 두고, 수정/커밋/알람 테스트를 진행해야 함
  3. 작업 일지를 잘 작성하는가 ?

## 작업 일지(journal.md) 작성 규칙
- 날짜 시간 (+09:00 처럼 타임존을 함께 표시), 작업 내용을 작성한다
- 과거 내용은 삭제하지 않고, 작업 내용을 누적 시킨다

## AI 지침
- AI 가 작업하는 내용은 .ai 폴더 하위에 기록, 참고 한다
- AI 가 생각을 하기 전에 .ai/prompts 하위 문서를 먼저 읽고 생각을 시작한다
  - 필요한 경우 md 파일을 수정한다
- AI 의 작업 결과물 중 저널, 결과 리포트 등은 .ai/llm_outs 폴더 하위에 기록한다
- PROMPT.md 파일은 AI 가 수정하지 않는다
- 코드 수정, 작성은 되도록 최소한의 코드를 사용한다
- 작업 목록은 미진행([ ]), 완료([x]), 진행중([/]) 표기를 사용하여 목록으로 관리한다
- 작업 목록에 일시중단([-]) pending 작업은 **절대** 작업 진행을 하지 않는다
- 작업을 1개 완료하면 작업 일지를 반드시 작성한다
- 작업 1개마다 검증 작업 1개를 되도록이면 같이 진행한다
- 작업 내용에 **'스크린샷 참고'** 가 있을 경우, .ai/assets/bugs 폴더를 우선적으로 검색해서 스크린샷 이미지를 참고 한다
- gemini cli 에서 커밋하라는 명령을 받거나, gemini-cli 가 커밋하는 경우 author 를 Gemini CLI <gemini-cli@example.com> 로 기록한다
- LLM 은 작업이 끝난 후 "ㅇㅇㅇ 을 할까요?" 처럼 추가 질문을 하지 않는다
- 작업 1개를 완료 한 후 작업 완료를 알리는 알람 스크립트를 실행한다
  - 스크립트 명령어 : `sh ~/gemini_alert.sh "작업내용..."`
